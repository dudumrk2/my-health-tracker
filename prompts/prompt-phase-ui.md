קרא קודם את `CLAUDE.md`, `screens-spec.md`, ו-`HLD-health-tracker.md`. ה-`screens-spec.md` הוא מקור האמת לעיצוב — עקוב אחריו בדיוק (מסכים, רכיבים, states, צבעים, RTL).

**המשימה:** בנה את שכבת ה-UI המלאה של האפליקציה ב-Jetpack Compose, עם נתוני דמה (mock). זהו שלב UI-first — אנחנו רוצים לראות ולהרגיש את כל המסכים לפני חיבור הלוגיקה האמיתית.

## עקרונות

1. **Mock data בלבד** — צור שכבת `FakeRepository` / mock data providers שמחזירים נתונים סטטיים מציאותיים (צעדים, שינה, ארוחות, תובנות AI לדוגמה). אל תתחבר ל-Firebase / Health Connect / Gemini.
2. **ארכיטקטורה נכונה כבר עכשיו** — MVVM, ViewModel לכל מסך, State hoisting, Composables ניתנים ל-preview. כך שחיבור הלוגיקה האמיתית בהמשך יהיה החלפת ה-mock ב-repository אמיתי, בלי לשכתב UI.
3. **כל הנתונים דרך ViewModel** — גם ה-mock. אסור hardcode בתוך Composable; הנתונים זורמים מ-ViewModel דרך State.

## מסכים לבנות (לפי screens-spec.md)

1. **כניסה** — לוגו, טאגליין, כפתור "המשך עם Google" (mock — לחיצה מעבירה הלאה).
2. **פרטים אישיים** — שנת לידה, מין (זכר/נקבה/אחר), משקל, גובה. ולידציה. כפתור סיום.
3. **דשבורד** (טאב) — סרגל עליון (שלום + רענון + פרופיל); כרטיסים: תובנת AI מאוחדת, פעילות יומית + משפט המלצה, שינה + שלבים + המלצה, סיכום אוכל שבועי, מדדי גוף + גרף מגמה + כפתור הוספה.
4. **פעילות** (טאב) — בורר תאריך עליון + רענון; טבעת צעדים מול יעד; כרטיס שינה; רשימת אימונים (HC + ידני עם תווית); כפתור הוספת אימון.
5. **אוכל** (טאב) — בורר תאריך + רענון; כרטיס המלצה יומית; סיכום תזונתי (קלוריות + מאקרו עם ברים צבעוניים); יומן ארוחות; יומן מים + כפתור הוספת מים מהיר; כפתור הוספת ארוחה.
6. **הוספת אימון** — בחירת סוג (רשת אייקונים), **משך בדקות**, תאריך+שעה (ברירת מחדל: עכשיו). כפתור שמירה.
7. **הוספת ארוחה** — כל ה-states: בחירת קלט (טקסט/תמונה) → טעינה ("מנתח...") → תוצאה הניתנת לעריכה (פריטים + מאקרו) → שגיאה עם fallback ידני. (ה-mock: לחיצה מדמה ניתוח עם השהיה קצרה ומחזירה תוצאה לדוגמה.)
8. **הוספת מדידת גוף** — תאריך (ברירת מחדל: היום), משקל, **היקף מותן, היקף ירכיים**, הערה (כל השדות אופציונליים). אופציונלי: איור גוף עם נקודות מדידה. כפתור שמירה.

## דרישות עיצוב (קריטי)

- **RTL מלא** — `LayoutDirection.Rtl`, כולל חצי בורר תאריכים, אייקונים וכפתורים. בדוק שכל מסך נראה נכון ב-RTL.
- **Light + Dark theme** — הגדר Material 3 `ColorScheme` לשני המצבים לפי הצבעים ב-screens-spec (accent בהיר `#0F6E56`, כהה `#5DCAA5`; רקעים וטקסט לפי המפרט). כל מסך חייב להיראות תקין בשני המצבים.
- **צבעי מאקרו עקביים** — חלבון=כחול, פחמימות=ענבר, שומן=אדום-ורוד, בכל המסכים.
- **תאריך ברירת מחדל = היום** (לא תאריך קשיח).
- Bottom Navigation עם 3 טאבים; מסכי ההוספה הם מודאליים/נפרדים ללא טאב-בר.
- `@Preview` Composables לכל מסך, בשני ה-themes.
